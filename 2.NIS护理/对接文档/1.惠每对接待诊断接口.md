# 惠每对接护理NIS待诊断接口

# 护理评估接口（WebToPIO）对接文档（祈福版本）

## 一、接口基本信息

| 项目     | 详情                                                         |
| -------- | ------------------------------------------------------------ |
| 接口类型 | Web接口                                                      |
| 接口地址 | `http://10.161.211.71:11004/Neusoft.NIS.NursingProcess.BllFacade.IPDAFacade.svc` |
| 方法名   | `WebToPIO(string json)`                                      |

## 二、请求参数说明

接口通过 **JSON字符串** 传递参数，JSON结构包含患者基本信息及评估项列表，参数详细说明如下：

| 字段名           | 类型     | 注释         | 是否必填 | 说明                                         |
| ---------------- | -------- | ------------ | -------- | -------------------------------------------- |
| `PatientID`      | string   | 患者流水号   | 是       | 患者唯一标识                                 |
| `PatientName`    | string   | 患者姓名     | 否       | 示例中为 `null`，可传空或实际姓名            |
| `EmrAssessList`  | array    | 评估项列表   | 是       | 评估项的集合，每个元素为一个评估项对象       |
| （评估项子字段） |          |              |          |                                              |
| `Guid`           | string   | 评估项GUID   | 否       | 评估项唯一标识                               |
| `AssItemId`      | string   | 评估术语ID   | 是       | 评估术语的唯一ID                             |
| `AssItemName`    | string   | 评估术语名称 | 是       | 评估术语的名称                               |
| `AssDate`        | datetime | 评估时间     | 是       | 格式示例：`2022-10-13T17:10:05.772241+08:00` |
| `AssResult`      | string   | 评分         | 是       | 评估结果的分数或描述                         |
| `AssevalorCode`  | string   | 评估人ID     | 是       | 评估人员的唯一标识                           |
| `AssevalorName`  | string   | 评估人姓名   | 是       | 评估人员的姓名                               |
| `FileDataId`     | string   | 文件数据ID   | 否       | 关联的文件数据标识                           |
| `DiagFrom`       | string   | 数据来源     | 是       | 数据的来源系统或模块                         |

## 四、返回值说明

接口返回值为 **整数**，含义如下：

| 返回值 | 含义                   | 说明                             |
| ------ | ---------------------- | -------------------------------- |
| `-1`   | 调用失败               | 接口调用过程中出现异常           |
| `0`    | 调用成功，未生成待诊断 | 数据接收成功，但未触发待诊断生成 |
| `1`    | 调用成功，生成待诊断   | 数据接收成功，且成功生成待诊断   |
