## 2.行业树术语[]

> ==SQL脚本==
>
> ```sql
> -- 行业树术语
> SELECT * FROM NIS_CBK_NAS_COMMON_TREE t1 
> where t1.id ='02e86e67-64a6-426d-967d-0d52fdef92c8' ;
> 
> -- 支持下钻查出子元素 
> SELECT 
>     id AS "ID",
>     parent_id AS "父ID",
>     -- 使用更直观的层级标识：添加前缀符号和缩进
>     CASE LEVEL
>         WHEN 1 THEN '├─ ' || name
>         WHEN 2 THEN '│  ├─ ' || name
>         WHEN 3 THEN '│  │  ├─ ' || name
>         WHEN 4 THEN '│  │  │  ├─ ' || name
>         ELSE LPAD(' ', (LEVEL-1)*4) || '└─ ' || name
>     END AS "层级名称",
>     title AS "标题",
>     control_type AS "控件类型",
>     unit AS "单位",
>     DECODE(is_readonly, '1', '只读', '可编辑') AS "编辑状态",
>     DECODE(is_hide, '1', '隐藏', '显示') AS "显示状态",
>     DECODE(valid_state, '1', '有效', '0', '无效', '未知') AS "有效性",
>     sort_no AS "排序号",
>     risk_level AS "风险等级",
>     -- 更清晰的层级数字显示
>     'L' || LEVEL AS "层级"
> FROM NIS_CBK_NAS_COMMON_TREE
> START WITH id = '9afa1a1a-857d-4535-9795-f1b31b86ee70'
> CONNECT BY PRIOR id = parent_id
> ORDER SIBLINGS BY sort_no;
> ```

## 3.护理患者评估

主要分为 `Nis_Nas_Record`和``Nis_Nas_Record_Item`，主要根据`Nis_Nas_Record.Id`和`Nis_Nas_Record_Item.record_id`进行连接

> 1. ==SQL脚本获取儿童跌倒风险评估单数据==
>
> ```sql
> -- 查询量表或评估表具体值得内容
> SELECT nsi.item_name||':'||nsi.item_content as item_info,  -- 拼接的数据
>            nr.create_date,
>            nr.template_name,
>            nr.content
>     FROM Nis_Nas_Record_Item nsi, Nis_Nas_Record nr
>     WHERE nsi.record_id = nr.id 
>     AND nsi.inpatient_code = 'ZY0010000001138' -- 患者住院流水号
>     AND nsi.common_tree_id = '5ffb0f49-a770-4e53-9a33-fd12581438c7' -- 行业树种想要查询的结果
>     AND nsi.valid_state = 1 -- 是否填写有效
>     ORDER BY nr.create_date DESC -- 降序
> 
> ```
>
> ![image-20250612143346120](https://gitee.com/HavertzPlatform/worker-picgo/raw/master/20250612143346268.png)
>
> 2. ==获取新生儿疼痛评估单==
>
> ```sql
> SELECT nsi.common_tree_id,
>            nsi.item_name || ':' || nsi.item_content as item_info,
>            nr.create_date,
>            nr.template_name,
>            nr.content
> FROM Nis_Nas_Record_Item nsi, Nis_Nas_Record nr   
> WHERE nsi.record_id = nr.id
> AND nsi.inpatient_code = 'ZY0010000001138' -- 住院流水号
> AND nsi.common_tree_id in
>            ('0a101772-a79c-4c0c-9ec4-bec01ebe1dab', -- 疼痛程度:重度疼痛；

>            ) 
> AND nsi.valid_state = 1 -- 是否有效
> ORDER BY nr.create_date DESC
> FETCH FIRST 2 ROW ONLY;
> ```

## 24.NIS配置产程图

```sql
写护理记录时同步写产程图验证步骤：
1.查询产程图模板ID（select ID from nsrt_input_template a where a.name = 'NQ产程图';），以363举例

2.查询需要同步到产程图的护理记录模板（select rowid,a.* from nsrt_input_template a where name = '产前BALABALA记录' and a.is_valid=1 and a.is_assign=1;），然后在CUSTOM_CODE字段中把上一步查到的模板ID写进去

3.替换dll并测试。写某条护理记录时，如果内容中包括血压、胎心、宫口扩张、先露高低这四项中的至少一项，则同步写一条产程图的记录，不包括则不写，比如护理记录里写了血氧饱和度，这个项目在产程图里是没有的，则不往产程图中写。
修改护理记录时，如果原来的项目是血氧饱和度和胎心，修改后把胎心去掉了只保留了血氧饱和度，则同一时间点的那条产程图记录也会删除。
删除护理记录时，如果删除的这条记录包括了产程图中的项目，则同一时间点的那条产程图记录也会删除。

       CASE
         WHEN MAX(r.bed_stand + r.bed_add) > 0 
         THEN TO_CHAR(ROUND((MAX(r.bed_stand) - MIN(r.bed_free)) /MAX(r.bed_stand + r.bed_add) * 100,2),'FM99999990.00') || '%'
         ELSE '0.00%'
       END AS 病床使用率,
       NVL(ts.rb_count, 0) AS 病房周转次数,
       NVL(ahd.avg_hospital_days, 0) AS 出院患者平均住院天数,
       SUM(r.in_normal + r.in_emergency + r.in_return) AS 入院患者数,
       SUM(r.in_transfer) AS 转入患者数,
       NVL(ahd.discharged_count, 0) AS 出院患者人数,
       SUM(r.out_transfer) AS 转出患者数,
       SUM(r.dead_in24 + r.dead_out24) AS 死亡患者数,
       -- NVL(ahd.total_hospital_days, 0) AS 住院总天数,
       SUM(r.out_normal + r.out_transfer + r.out_withdrawal) AS 出院患者数,      
       MAX(r.end_num) AS 在院患者数,   
       NVL(cp.critical_count, 0) AS 病危患者数,
       NVL(sp.serious_count, 0) AS 病重患者数,       
       NVL(psp.level_1, 0) AS 一级手术,
       NVL(ssp.level_2, 0) AS 二级手术,
       NVL(tsp.level_3, 0) AS 三级手术,
       NVL(qsp.level_4, 0) AS 四级手术,
       NVL(tosp.oper_num,0)AS 明日手术,
       NVL(dpc.Birthtotal,0)AS 分娩患者数,
       NVL(pcl.FirstLevelCare,0) AS 一级护理,
       NVL(scl.SECONDLEVELCARE,0) AS 二级护理,
       NVL(tcl.THIRDLEVELCARE,0) AS 三级护理,
       NVL(spcl.SPECIALLEVELCARE,0) AS 特级护理,
	   NVL(pupc.high_risk_num,0) AS 压疮患者数,
	   NVL(puhr.risk_item_count,0) AS 压疮高风险患者数
  FROM met_cas_inpatientdayreport r
  LEFT JOIN turnover_stats ts
    ON r.nurse_cell_code = ts.nurse_cell_code
  LEFT JOIN avg_hospital_days_stats ahd
    ON r.nurse_cell_code = ahd.nurse_cell_code
  LEFT JOIN critical_patients cp
    ON r.nurse_cell_code = cp.NurseCellCode
  LEFT JOIN serious_patients sp
    ON r.nurse_cell_code = sp.NurseCellCode
  LEFT JOIN primary_surgery_patients psp
    ON r.nurse_cell_code = psp.dept_code
  LEFT JOIN  secondary_surgery_patients ssp
    ON r.nurse_cell_code = ssp.dept_code  
  LEFT JOIN  tertiary_surgery_patients tsp
    ON r.nurse_cell_code = tsp.dept_code     
  LEFT JOIN  quaternary_surgery_patients qsp  
    ON r.nurse_cell_code = qsp.dept_code
  LEFT JOIN tomorrows_surgery_patients tosp
    ON r.nurse_cell_code = tosp.dept_code 
  LEFT JOIN delivery_patient_count dpc 
    ON  r.nurse_cell_code = dpc.Nursecode
  LEFT JOIN primary_care_level pcl
    ON  r.nurse_cell_code = pcl.DEPARTMENTCODE
  LEFT JOIN secondary_care_level scl   
    ON  r.nurse_cell_code = scl.DEPARTMENTCODE 
  LEFT  JOIN tertiary_care_level tcl    
    ON r.nurse_cell_code = tcl.DEPARTMENTCODE
  LEFT JOIN special_care_level spcl
    ON r.nurse_cell_code = spcl.DEPARTMENTCODE
	LEFT JOIN pressure_ulcer_patient_count pupc
	  ON  r.nurse_cell_code = pupc.nurse_code
	LEFT JOIN pressure_ulcer_highrisk puhr
	   ON  r.nurse_cell_code = puhr.dept_code       
 WHERE r.date_stat = TO_DATE('2025-12-03', 'YYYY-MM-DD')
   AND r.state = '1'
   AND r.nurse_cell_code IS NOT NULL
   AND r.nurse_cell_code <> 'null'
 GROUP BY r.date_stat,
          r.nurse_cell_code,
          ts.rb_count,
          ahd.avg_hospital_days,
          ahd.discharged_count,
          ahd.total_hospital_days,
          cp.critical_count,
          sp.serious_count,
          psp.level_1,
          ssp.level_2,
          tsp.level_3,
          qsp.level_4,
          tosp.oper_num,
          dpc.Birthtotal,
          pcl.FIRSTLEVELCARE,
          scl.SECONDLEVELCARE,
          tcl.THIRDLEVELCARE,
          spcl.SPECIALLEVELCARE,
		  pupc.high_risk_num,
		  puhr.risk_item_count
```
