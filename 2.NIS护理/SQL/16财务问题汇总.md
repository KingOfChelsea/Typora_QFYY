## 1.住院订单头表索引唯一性报错（祈福）

- 问题**如图3**所示，跟SQL排查发现，SQL语句去更新`fin_ipb_order`的数据，因为索引的唯一性质（如**图1**所示），把其他的订单的ORDER_ID前面加入`B`,**如图2**所示，

```sql
update fin_ipb_order
   set invoice_no       = 'E2026062150599',
       balance_no       = '1',
       balance_oper_code = 'admin',
       balance_date     = to_date('2026-06-21 20:27:19', 'yyyy-MM-dd hh24:mi:ss'),
       begin_date       = case when begin_date > to_date('2000-01-01 00:00:00','yyyy-MM-dd hh24:mi:ss') then begin_date else to_Date('2026-06-20 16:26:00', 'yyyy-MM-dd hh24:mi:ss') end,
       end_date         = case when end_date > to_date('2000-01-01 00:00:00','yyyy-MM-dd hh24:mi:ss') then end_date else to_Date('2026-06-21 19:40:59', 'yyyy-MM-dd hh24:mi:ss') end,
       order_state      = '1'
 where order_id = '2026542120500024875'
   and inpatient_no = 'ZY001B100519003'
   and order_state = '0'

Error occurred: [1] (ORA-00001: 违反唯一约束条件 (HIS50.PK_FIN_IPB_ORDER)
)
```
